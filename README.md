# Java Architect / Principal Mastery Roadmap

> From Fresher to Java Architect -- a structured, hands-on learning path.

## How to Use This Roadmap

1. Work through **Part 1 (Core Java)** first, then **Part 2 (Spring Framework)**.
2. Each phase has its own `README.md` with topics, references, and exercises.
3. Exercises come with starter code -- implement the `// TODO` sections.
4. Use the **progress tracker** below to mark your progress.
5. **Do not skip phases** -- later phases build on earlier ones.

## Time Commitment

| Pace | Daily Hours | Part 1 (Core Java) | Part 2 (Spring) | Total |
|------|-------------|---------------------|------------------|-------|
| Standard | 2-3h | ~18 weeks | ~14 weeks | ~32 weeks |
| Intensive | 4-5h | ~10 weeks | ~8 weeks | ~18 weeks |

## Recommended Books (Priority Order)

| Book | Author | Covers |
|------|--------|--------|
| Effective Java (3rd Ed.) | Joshua Bloch | Best practices, idioms |
| Java Concurrency in Practice | Brian Goetz | Multithreading mastery |
| Head First Design Patterns (2nd Ed.) | Freeman & Robson | Pattern thinking |
| Modern Java in Action | Urma, Fusco, Mycroft | Lambdas, streams, modern features |
| Clean Architecture | Robert C. Martin | Architecture principles |
| Domain-Driven Design | Eric Evans | DDD for architects |
| Spring in Action (6th Ed.) | Craig Walls | Spring ecosystem |
| Java Performance (2nd Ed.) | Scott Oaks | JVM tuning |

## Tools You Will Need

- **JDK 21+** (LTS) -- [Adoptium](https://adoptium.net/)
- **Maven 3.9+** -- [maven.apache.org](https://maven.apache.org/)
- **IDE** -- IntelliJ IDEA (recommended) or VS Code with Java Extension Pack
- **Git** -- version control for your exercises
- **Docker** -- needed for Spring phases (databases, messaging)
