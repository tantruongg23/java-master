# Part 1: Core Java Mastery

> Master the Java language itself before touching any framework.

## Why Core Java First?

Frameworks come and go, but deep Java knowledge is permanent. As an architect, you will:
- Debug framework internals when things break
- Make technology decisions that require understanding trade-offs at the language/JVM level
- Design systems that outlive any single framework version
- Review and mentor others on foundational code quality

## Phases

### Foundation Layer (Phases 01-04) -- ~7-9 weeks

These phases cover what every Java developer must know cold.

| Phase | Topic | Key Outcome |
|-------|-------|-------------|
| [01](phase-01-fundamentals/) | Fundamentals (Speed Run) | Fluent in syntax, String internals, memory basics |
| [02](phase-02-oop-deep-dive/) | OOP Deep Dive | SOLID principles, design thinking, polymorphism mastery |
| [03](phase-03-collections-generics/) | Collections & Generics | Know which data structure to use and why, type-safe APIs |
| [04](phase-04-exception-io-nio/) | Exception Handling, I/O, NIO | Robust error handling, efficient file processing |

### Advanced Layer (Phases 05-07) -- ~8-11 weeks

This is what separates senior developers from juniors.

| Phase | Topic | Key Outcome |
|-------|-------|-------------|
| [05](phase-05-concurrency/) | Concurrency & Multithreading | Thread-safe code, performance via parallelism |
| [06](phase-06-jvm-internals/) | JVM Internals & Performance | Tune JVM, diagnose memory leaks, understand GC |
| [07](phase-07-design-patterns/) | Design Patterns | Recognize and apply patterns in real systems |

### Architect Layer (Phases 08-11) -- ~9-12 weeks

Architect-level thinking: modern idioms, quality, and system design.

| Phase | Topic | Key Outcome |
|-------|-------|-------------|
| [08](phase-08-functional-programming/) | Functional Programming | Streams, lambdas, declarative data pipelines |
| [09](phase-09-modern-java/) | Modern Java (9-21+) | Leverage latest language features effectively |
| [10](phase-10-testing-quality/) | Testing & Code Quality | TDD, comprehensive test strategies, quality gates |
| [11](phase-11-architecture-patterns/) | Architecture Patterns | DDD, hexagonal, CQRS, microservices design |

## How Each Phase Works

1. **Read** the phase README -- understand topics and objectives
2. **Study** the referenced materials (books, docs, articles)
3. **Code** the exercises -- real-world problems with starter code
4. **Self-assess** using the checklist at the end of each phase
5. **Move on** only when you can explain each topic to someone else

## Prerequisites

- JDK 21+ installed
- Maven 3.9+ installed
- An IDE (IntelliJ IDEA recommended)
- Basic familiarity with Java syntax (variables, if/else, loops, methods)
