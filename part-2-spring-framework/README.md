# Part 2: Spring Framework Mastery

> Apply your Core Java expertise to the industry-standard enterprise framework.

## Prerequisites

You **must** complete Part 1 (Core Java) before starting here. Spring heavily relies on:
- OOP and interfaces (Phase 02)
- Generics and collections (Phase 03)
- Concurrency (Phase 05) -- for async processing, WebFlux
- Design patterns (Phase 07) -- Spring IS patterns (Factory, Proxy, Template, Observer)
- Functional programming (Phase 08) -- for WebFlux, Stream-based queries
- Architecture patterns (Phase 11) -- for microservices design

## Phases

| Phase | Topic | Key Outcome |
|-------|-------|-------------|
| [01](phase-01-spring-core-ioc/) | Spring Core & IoC | Understand DI, beans, AOP -- the foundation of everything Spring |
| [02](phase-02-spring-boot/) | Spring Boot | Auto-configuration, production-ready apps, custom starters |
| [03](phase-03-spring-data-jpa/) | Spring Data & JPA | Database access, transactions, query optimization |
| [04](phase-04-spring-web-rest/) | Spring Web & REST | RESTful APIs, validation, error handling, WebSocket |
| [05](phase-05-spring-security/) | Spring Security | Authentication, authorization, JWT, OAuth2 |
| [06](phase-06-spring-cloud-microservices/) | Spring Cloud & Microservices | Distributed systems, service mesh, messaging |
| [07](phase-07-advanced-spring/) | Advanced Spring | Batch, reactive, observability, performance |
| -- | [Capstone Project](capstone-project/) | Full production-grade system combining everything |

## Tools You Will Need (In Addition to Part 1)

- **Docker & Docker Compose** -- for databases, message brokers, etc.
- **Postman or HTTPie** -- for API testing
- **PostgreSQL** -- primary database (via Docker)
- **Redis** -- caching exercises (via Docker)
- **Apache Kafka** -- messaging exercises (via Docker)
